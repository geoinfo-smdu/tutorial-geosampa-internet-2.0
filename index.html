Tutorial GeoSampa Internet

Este tutorial auxilia no uso das ferramentas disponíveis e na navegação pelos mapas e camadas de dados do GeoSampa Internet, que pode ser acessado pela população em geral.

O GeoSampa Internet passou por uma reformulação da Interface de navegação. Este tutorial é relacionado à essa nova versão. Nos próximos meses, a versão antiga e nova do GeoSampa irão coexistir para que façamos uma transição tranquila. A seguir, deixamos os links de acesso das diferentes versões.

Especificações técnicas

Sistema de Referência Cartográfica

No GeoSampa, as informações são disponibilizadas em diversos sistemas de referência cartográfica, como SIRGAS2000 e WGS84. A projeção UTM/SIRGAS2000 (EPGS: 31983) é a referência para o Sistema Geodésico Brasileiro (SGB) e para o Sistema Cartográfico Nacional (SCN). A seguir, deixamos os sistemas de referência cartográfica que podem ser encontrados no GeoSampa:

Datum Projeção EPSG**
SIRGAS 2000 UTM* 23 S 31983
SIRGAS 2000 GEOGRÁFICA 4674
WGS 84 UTM* 23 S 32723

*UTM: Universal Transversa de Mercator - Sistema de coordenadas cartesianas

**A EPSG vem do nome do Grupo Europeu de Pesquisa Petrolífera (European Petroleum Survey Group), entidade que organizou por meio de códigos numéricos os Sistemas de Referência de Coordenadas (SRC) do mundo.

Dicas e instruções importantes

Verifique se seu bloqueador de “pop-up” está desativado. Caso contrário, alguns downloads podem ser bloqueados.

Para evitar possíveis erros, sugerimos realizar periodicamente a limpeza do cache do seu navegador. Para isso, basta acionar o atalho Ctrl + Shift + Del.

Disponibilização dos dados

Webservice - Serviços de mapa

Webservice é o método utilizado para transferir dados através de URL/links entre diferentes plataformas. Os serviços web do GeoSampa estão disponíveis em dois formatos: WMS (Web Map Service) e WFS (Web Feature Service).

WMS é um serviço de imagem que serve principalmente como georreferência, não sendo possível nenhuma edição ou cruzamento espacial.

O serviço WFS disponibiliza o acesso e manipulação dos dados geográficos a partir de vetores, permitindo operações espaciais.

O GeoSampa, por meio de seus serviços online (WMS e WFS), possibilita que os usuários acessem os dados disponíveis em ambientes fora do Mapa Digital da Cidade.

Para acessar os dados do GeoSampa por meio de serviço on-line, utilize os links abaixo:

Tipo de serviço Descrição Link
WMS - Camadas Serviço WMS para as camadas do GeoSampa http://wms.GeoSampa.prodam/geoserver/geoportal/wms
WMS - Mapas Base Serviço WMS para os mosaicos de ortofotos e levantamentos aéreos (como o SARA Brasil 1930 e VASP Cruzeiro 1954) http://raster.GeoSampa.prodam/geoserver/geoportal/wms
WFS - Camadas Serviço WFS para as camadas do GeoSampa http://wfs.GeoSampa.prodam/geoserver/geoportal/wfs

Download de arquivos

O conteúdo do GeoSampa que está disponível para download se apresenta em diversos formatos. Para baixar os dados de uma camada, clique no botão Saiba mais sobre a camada e, depois, no botão

Será aberta uma caixa de diálogo apresentando as opções de formatos disponível para download. Para fazer o download, basta clicar no botão ao lado do formato escolhido.

Em alguns casos, por conta do tamanho dos arquivos, os dados disponíveis para download de algumas camadas são particionados por distrito. É o caso, por exemplo, das camadas 'Edificações 2D' e 'Lotes'.

O GeoSampa trabalha com arquivos em diversos formatos, variando de acordo com a natureza e a forma de construção dos dados. O quadro a seguir lista os formatos disponíveis no GeoSampa (para download e/ou upload):

PDF* Portable Document Format: arquivo para visualização, sem possibilidade de edição.
DXF* Drawing Exchange Format: arquivo vetorial, em código aberto, usado para compartilhamento universal em formato CAD. No GeoSampa é usado para disponibilizar o mapeamento altimétrico e planialtimétrico do Mapa Digital da Cidade (MDC). Os arquivos estão divididos por folhas do Sistema Cartográfico Metropolitano (SCM), representadas como quadrículas no mapa. Os dados disponíveis na projeção em que foram construídos: SAD69 (EPSG 5533).
SHP Shapefile: formato popular de arquivo contendo dados geoespaciais em forma de vetor e usado por Sistemas de Informações Geográficas.
KMZ* Keyhole Markup Zipped: linguagem baseada em XML e serve para visualização mapas em 2D e navegadores terrestres em 3D. É o padrão utilizado pelo Google Earth.
XLS | CSV* Excel spreadsheet | Comma-Separated Values: Planilhas de dados e séries históricas.
GPKG GeoPackage: formato de dados abertos, não proprietário, baseado em padrões para sistema de informações geográficas e implementado como um banco de dados.
JPEG | JPG* Joint Photographic Experts Group: formato de imagem em arquivo aberto, comprimido para redução do tamanho. Geralmente é acompanhado de arquivo .aux com a informações de localização geográfica para ser utilizado em softwares de geoprocessamento.
GeoJSON GeoJSON é um formato de intercâmbio de dados geoespaciais padrão aberto que representa feições geográficas simples e seus atributos não espaciais. Baseado em JavaScript Object Notation (JSON), GeoJSON é um formato para codificar uma variedade de estruturas de dados geográficos.
GPX** GPS Exchange Format é um formato aberto, baseado em XML que armazena dados geográficos em pontos e linhas.
TopoJSON** Extensão do GeoJSON que reconhece topologia e codifica arcos compartilhados para reduzir o tamanho do arquivo, especialmente para bordas de polígonos.
IGC Files** São registros digitais padronizados de voos, contendo dados de GPS, altitude e tempo.

* Disponíveis apenas para download de algumas camadas.

** Disponíveis apenas para upload.

Conceitos

Dados abertos

Os dados em formato aberto permitem que qualquer cidadão utilize variados softwares para adaptá-los às suas necessidades. Além disso, estas informações contribuem para a transparência e o processo participativo, bem como para a criação de novas aplicações colaborativas.

Metadados

Os metadados são as informações sobre cada uma das camadas e têm como objetivo proporcionar a correta utilização e interpretação dos dados, indicando, por exemplo, denominação, descrição do conteúdo, escala, tipo de arquivo, órgão responsável, periodicidade de atualização, entre outras informações.

Clicando no botão Metadados, localizado no canto superior direito do GeoSampa, você acessa o Catálogo de Metadados classificado por tópicos. Para algumas bases, também está disponível o dicionário de dados disponível em planilha.

Ao acessar o Catálogo de Metadados, você encontrará uma barra de pesquisa. Nela, você pode pesquisar pelo nome de uma camada ou por um termo. Você pode também fazer a consulta a partir dos Tópicos, marcadores criados para agrupar camadas com tema semelhante.

Ao acessar o metadado de uma camada, você encontrará, no mínimo, os seguintes itens preenchidos:

  • Título é o nome da camada. É o mesmo nome que ela recebe no Menu de Camadas, no GeoSampa.
  • Resumo é a descrição breve do conteúdo, propósito e características da camada.
  • Datas contam com informações temporais relevantes, como criação, publicação e última atualização.
  • Contatos do recurso identificam pessoas ou organizações responsáveis pelo recurso, incluindo meios de contato.
  • Frequência de atualização trata da periodicidade com que a camada é atualizada (ex.: anual, mensal, contínua).
  • Tipo de representação é a forma de representação espacial dos dados, como vetor, raster, grade ou texto.
  • Sistema de referência e coordenadas trata do SRC com que a camada foi criada.
  • Linhagem diz respeito ao histórico de origem, métodos de produção, processamento e qualidade geoespacial.

Dicionário de dados

Um dicionário de dados é um descritivo das informações contidas nas camadas e conjunto de dados, fornecendo definições sobre os elementos da base de dados, como variáveis, formatos, valores permitidos, origens e relacionamentos.

Ele funciona como um glossário técnico, garantindo que todos os envolvidos tenham uma compreensão uniforme dos dados, facilitando a comunicação, a padronização do uso de variáveis e a criação de relatórios e sistemas mais confiáveis e reprodutíveis.

Este recurso está sendo disponibilizado gradualmente para as camadas.

Escala

Escala cartográfica é a proporção de redução da área da paisagem real para sua representação no mapa. Existem dois tipos de escalas cartográficas: numérica e gráfica.

  • Escala numérica expressa o valor em números. No caso de uma escala 1:501.405 (um para 501.405): significa que um centímetro no mapa corresponde a 501.405 centímetros (equivalente a 5,01 quilômetros) na paisagem real;
  • Escala gráfica indica a correspondência entre um segmento e uma distância na paisagem real, podendo ser expressa, no GeoSampa, em metros ou quilômetros.

Coordenadas

Coordenada plana é uma forma de localizar um ponto exato em uma superfície plana usando dois números, chamados de par ordenado (X, Y), a partir do sistema métrico. A caixa de coordenadas apresenta as coordenadas X e Y por onde o cursor do mouse estiver passando no mapa. Clicando na caixa de coordenadas, uma caixa de diálogo será aberta.

Nela, é possível alterar o sistema de projeção da caixa de coordenadas, bem como o número de casas decimais apresentadas.

Busca

Aspectos gerais

A pesquisa no GeoSampa está concentrada em uma única ferramenta, localizada no canto superior esquerdo da tela.

A busca textual foi desenvolvida com pesquisa fonética, de modo que o motor de busca codifica os nomes por sons (fonemas) e não apenas pela grafia exata. Assim, é possível encontrar registros mesmo com erros ortográficos, variações ou acentuação incorreta para logradouros, distritos, subprefeituras e outros.

Ao clicar nas Configurações das opções de busca, você encontrará todas as opções de pesquisa disponíveis. As pesquisas disponíveis para buscar no mapa do GeoSampa estão listadas abaixo:

Quando você pesquisar um termo na barra de pesquisa, o motor de busca promoverá a pesquisa em todas as bases listadas acima. Por padrão, sempre estarão habilitadas todas as opções de pesquisa. Você pode desabilitar as que julgar necessário.

Você também pode criar uma ordem de hierarquia para a pesquisa, habilitando a opção Parar se encontrado na consulta que deve ser priorizada. Assim, o motor de busca iniciará a pesquisa na primeira base habilitada - no caso da imagem acima, Logradouros - Município de São Paulo. Como a opção Parar se encontrado está marcada, caso o motor de busca encontre algum resultado na base de logradouros, a busca será interrompida. No entanto, caso nenhum resultado seja encontrado, será feita busca por subprefeitura. Como a opção Parar se encontrado está desmarcada, mesmo que algum resultado seja localizado na base de subprefeituras, o motor de busca fará a busca por distrito. Ao final, serão apresentados todos os resultados encontrados. Você também pode habilitar a função Permitir múltiplos resultados de pesquisa no mapa ao mesmo tempo. Assim, o motor de busca pesquisará o termo em todas as pesquisas disponíveis e apresentará todos os retornos na visualização do mapa para que você possa escolher qual resultado te atende melhor.

Logradouros

Nestes casos, a busca é realizada em base de logradouros oficiais e não oficiais do município.

A pesquisa nesta camada pode se dar pelo nome do logradouro (p.ex. Avenida Paulista, Rua São Bento, Praça da Sé, etc.) ou pelo Código do Logradouro (CODLOG). A pesquisa pelo nome do logradouro é fonética e permite encontrar os registros desejados mesmo com erros ortográficos, variações ou acentuação incorreta.

Na pesquisa por endereço, quando é inserido apenas o nome do logradouro, a pesquisa retornará uma linha, que representa toda a extensão do logradouro.

Se a pesquisa for feita buscando pelo nome do logradouro e o número de porta, retornará como um ponto no mapa.

Se o ponto estiver dentro da quadra fiscal, significa que há um lote fiscal com o mesmo número de porta que foi pesquisado. Caso retorne um ponto no meio do leito do logradouro, significa que não foi encontrado um lote fiscal com o mesmo número de porta pesquisado, retornando ponto no eixo da via que representa o local mais próximo ao número pesquisado.

Setor.Quadra.Lote

Pesquisa nas camadas de Setor, Quadra e Lote Fiscais. A camada Setor representa a maior unidade territorial do cadastro fiscal do município. A camada Quadra representa a segunda unidade territorial do cadastro imobiliário fiscal do município. A camada Lotes é a representação cartográfica oficial do Cadastro Imobiliário Fiscal da Secretaria da Fazenda (SF) e a menor unidade de representação do cadastro fiscal.

A pesquisa pode ser feita:

  • Apenas pelo número do Setor, composto por 3 dígitos, que variam entre 001 e 310 (p.ex. 005)

  • Pelos números de Setor e Quadra, composto por 6 dígitos, sendo que a pesquisa por Setor.Quadra deve utilizar o ponto como separador numérico (p.ex. 123.123)

  • Pelos números de Setor, Quadra e Lote, composto por 10 dígitos, sendo que a pesquisa por Setor.Quadra.Lote deve utilizar o ponto como separador numérico (p.ex. 123.123.0001)

Coordenadas

Latitude / longitude - decimais

As coordenadas decimais são a forma mais simples e moderna de indicar um ponto exato na Terra, usando apenas números decimais (com vírgula ou ponto) em vez de graus, minutos e segundos.

Para essa pesquisa, você deve usar o ponto separador como separador numérico e a vírgula como separador entre a latitude e longitude (p.ex. -23.548173247112523, -46.63345328512641).

Latitude / longitude - graus, minutos e segundos

Coordenadas em Graus, Minutos e Segundos (GMS) localizam qualquer ponto na Terra dividindo o globo em 360°.

Para essa pesquisa, você deve usar os símbolos º para marcador de grau, ' para marcador de minuto, e '' para marcador de segundo. Considerando que serão pesquisados pontos na cidade de São Paulo, deve ser usada a letra S maiúscula (representando 'South') ao fim da coordenada da longitude e a letra W maiúscula (representando 'West') ao fim da coordenada da latitude (p.ex. 23°33'01.4"S, 46°38'02.2"W).

Coordenadas planas

Coordenada plana é uma forma de localizar um ponto exato em uma superfície plana usando dois números, chamados de par ordenado (X,Y), a partir do sistema métrico.

Para essa pesquisa, utilize o ponto como separador numérico e a vírgula como separador entre as coordenadas X e Y (p.ex. 333220.79268776, 7394600.1388575).

Pesquisa de endereços do Nominatim (OSM)

Nominatim é o motor de busca oficial por trás do OpenStreetMap (OSM). Ele funciona como um "tradutor" que transforma endereços ou nomes de locais em coordenadas geográficas.

O OpenStreetMap é um projeto colaborativo internacional que cria e disponibiliza um mapa-múndi livre e editável, funcionando de forma semelhante à Wikipédia. Construído por voluntários, utiliza dados de GPS, imagens de satélite e conhecimento local para fornecer informações geográficas gratuitas sob licença aberta.

Menu de ferramentas

À esquerda da tela está o menu de ferramentas, onde você encontrará funcionalidades que auxiliam na navegação e na consulta dos dados disponíveis.

Aproxima ou afasta. Ao alterar o zoom, a escala também é alterada. Cada camada possui um limite de escala de visualização, adequado à sua legibilidade. Caso necessário, deve-se aumentar a escala (Zoom +) até que a camada fique habilitada no menu de camadas, pois só é possível ligar a camada se o zoom estiver adequado à sua visualização. Por exemplo, a camada Lote pode ser visualizada, exclusivamente, em escala abaixo de 1:5.000.

Seguindo as convenções cartográficas, o mapa do GeoSampa sempre se inicia com a indicação de Norte para cima. A ferramenta Rotação permite que o usuário rotacione a visualização do mapa para melhor visualização dos dados.

Em caso de uso no computador, segure as teclas Alt + Shift e, com o botão esquerdo do mouse, faça a rotação no mapa. Clicando novamente no botão, a visualização retorna a seu estado inicial.

Em caso de uso no celular, clique no mapa do GeoSampa com dois dedos e faça a rotação.

Ao clicar nesse botão, será aberta caixa de opções para realizar medições cartográficas no mapa. Você poderá escolher medir uma área ou a distância entre pontos. Clicando no botão , será aberta caixa de diálogo em que você pode alterar as unidades de medida (metros ou jardas/milhas) e habilitar a exibição dos comprimentos de cada segmento de medição; e do comprimento e área total da medição realizada.

Para medir uma área, clique no botão . Para definir os vértices da área a ser calculada, clique com o botão esquerdo do mouse nos pontos desejados. Para definir o último vértice do polígono e finalizar o desenho, clique com o botão direito do mouse.

A ferramenta Desenhar Texto permite a inclusão de um texto ao mapa. O primeiro passo é definir o texto (que deve ser escrito no campo 'Texto') e sua aparência (demais campos) no menu lateral. Em seguida, para definir sua localização, clique com o botão esquerdo do mouse no local desejado.

As três últimas ferramentas são para a edição dos desenhos e textos adicionados.

É possível definir o nome do arquivo e seu formato. Vale reforçar que apenas o formato KML oferece suporte a dados de estilo e rótulos de texto.

Para alterar características da aparência e conteúdo das anotações, utilize a ferramenta Modificar Anotações . Depois de clicar no ícone indicado, selecione a anotação que será editada; e altere os parâmetros indicados no menu lateral.

O botão Excluir todas as Anotações restaura a visualização a sua condição original e exclui todas as anotações criadas.

A seção Consultas conta com ferramentas que permitem que o usuário acesse os dados de feições presentes no mapa.

O botão Consultar feições clicando no mapa permite que a consulta seja feita em um ponto escolhido pelo usuário. Assim, você pode obter os dados de uma feição específica clicando sobre ela, seja ela pertencente a uma camada ou criada por você com o controle de alterações.

Ao clicar sobre uma geometria, será habilitada uma caixa de diálogo com os dados disponíveis para a feição.

O botão Consultar feições desenhando um polígono permite que a consulta seja feita a partir de um polígono desenhado pelo usuário. Assim, você obtém os dados de todas as feições das camadas habilitadas que interseccionam o polígono desenhado.

Para definir os vértices do novo polígono, clique com o botão esquerdo do mouse nos pontos desejados. Para definir o último vértice do polígono e finalizar o desenho, clique com o botão direito do mouse. Em seguida, será habilitada a lista de resultados.

Clicando sobre o número de identificação na lista de resultados, você pode visualizar as informações disponíveis para a feição.

Para retornar à lista de resultados, clique sobre o botão Voltar para a lista de resultados.

O botão Consultar feições por um buffer permite que a consulta seja feita a partir de um buffer (área de influência radial) criado pelo usuário. Assim, você obtém os dados de todas as feições das camadas habilitadas que interseccionam o buffer desenhado.

Para definir sua localização, clique com o botão esquerdo do mouse para definir o centro do círculo. Uma caixa de diálogo será habilitada:

Digite o tamanho do buffer a ser criado; escolha a unidade de medida (metros, quilômetros ou milhas) e clique em Buscar. Será habilitada lista de resultados.

Clicando sobre o número de identificação na lista de resultados, você pode visualizar as informações disponíveis para a feição.

Para retornar à lista de resultados, clique sobre o botão Voltar para a lista de resultados.

Filtros

A ferramenta Filtros permite filtrar elementos de uma camada a partir do conteúdo de seus atributos. É possível, por exemplo, pesquisar um distrito pelo nome e/ou área. Ao clicar no ícone ao lado do nome da camada cujos registros serão filtrados (no caso, camada Distrito). Será habilitada uma janela como esta.

A caixa de filtros contém algumas condições a serem preenchidas pelo usuário para que sua busca seja direcionada. As condições são relacionadas às camadas selecionadas e aos seguintes operadores: 'igual a', 'diferente de', 'maior que', 'maior ou igual a', 'menor que', 'menor ou igual a', 'entre', 'é nulo'. Estes podem ser adicionados para buscas em textos ou em parâmetros numéricos e sua aplicação também é pré-definida pelo usuário, que escolhe se quer atender as buscas em 'todas', 'alguma' ou 'nenhuma' das condições definidas. Com isso, são diversas as combinações de filtros que podem ser aplicadas separado ou conjuntamente entre as camadas para a confecção de um projeto de visualização personalizado para as necessidades do usuário.

Pode-se, por exemplo, filtrar a geometria a partir de seu nome exato. Para isto, selecione o campo que conta com os dados dos nomes da camada (no caso, nm_distrito_municipal); selecione a opção Igual a; e insira o nome da geometria que se deseja filtrar (no caso, 'SÉ').

Clicando em Aplicar, o mapa irá enquadrar a(s) geometria(s) selecionadas. Caso a camada seja ligada, apenas a(s) geometria(s) filtradas serão reproduzidas.

Outra opção é filtrar nomes diferentes do especificado. Basta que, ao invés de Igual a, seja selecionada a opção Diferente de. Neste caso, foram filtradas todas as geometrias com nome diferente de 'SE'.

Neste caso, são apresentados todos os distritos de São Paulo, exceto a Sé.

Pode-se, também, filtrar a geometria a partir de parte de seu nome. Neste caso, por exemplo, foram filtrados todos os distritos com nome que contém a letra 'R' em sua grafia, independentemente da localização desta letra na palavra.

Existem algumas regras para esse tipo de busca:

  • A pesquisa diferencia letras minúsculas e maiúsculas;
  • Use * como coringa de qualquer sequência de caracteres;
  • Use . como coringa de um único caractere;
  • Use ! para escapar de caracteres especiais.

Portanto, para pesquisar distritos com nome que contém a letra R em sua grafia, é necessário fazer a pesquisa com *R*. Se a pesquisa for feita com R*, irá retornar apenas os distritos cujo nome começa com a letra R. Se for feita com *R, irá retornar apenas os distritos cujo nome termina com a letra R.

O . é usado da mesma forma, mas considera apenas um caractere antes ou depois do texto inserido. Portanto, se a pesquisa for feita com S., irá retornar o distrito da Sé. Se for feita com .APA, irá retornar o distrito da Lapa. Se for feita com .AUD., irá retornar o distrito da Saúde.

É possível, ainda, fazer filtros a partir de campos numéricos. Basta selecionar o campo, eleger a regra de comparação ('Igual a', 'Maior que', 'Menor que', entre outras) e inserir o número de referência. Neste caso, por exemplo, foram filtrados todos os distritos que contam com área maior que 150 quilômetros quadrados.

Como encontrar um número de IPTU (SQL)?

O código SQL (setor-quadra-lote) representa a identificação dos lotes no Cadastro Imobiliário Fiscal da Prefeitura Municipal de São Paulo. Como indicado na imagem a seguir, o código é composto a partir do número de identificação do setor, quadra e lote fiscal.

Este código corresponde ao número do contribuinte e pode ser localizado no boleto de pagamento do IPTU. Também é possível identificar o código SQL e outros dados dos lotes fiscais utilizando o GeoSampa.

O GeoSampa disponibiliza informações fiscais do lote na camada Lote, que é atualizada pela Secretaria da Fazenda. Esta camada substitui os antigos 'croquis fiscais', que não são atualizados desde junho de 2019.

Para facilitar a utilização do munícipe, as informações de SQL já aparecem concatenadas como código do contribuinte. Basta deixar a camada Lote ativa e clicar sobre alguma feição para visualizar as informações disponíveis. Será aberta uma caixa com o código de Setor.Quadra.Lote (SSS.QQQ.LLLL) e todas as informações pertencentes a ele nesta camada.

Para o caso de lotes com condomínios, ao abrir as informações sobre o lote, o Código do condomínio virá acompanhado do ícone de integração . Ao clicar nele, é aberta a aba de integrações com as informações sobre cada unidade condominial e suas respectivas informações.

Histórico

O Mapa Digital da Cidade (MDC) é um dos pilares fundamentais da infraestrutura de dados geoespaciais do Município de São Paulo, representando décadas de evolução tecnológica e institucional na produção, organização e disseminação de informações territoriais.

A trajetória começa ainda na década de 1970, com iniciativas pioneiras como o GEOLOG (1978), que já buscava digitalizar informações cartográficas. Em 2004, um marco importante foi a criação do MDC, resultado de um amplo levantamento aerofotogramétrico financiado pelo PMAT e pelo BNDES. Esse trabalho estruturou uma base cartográfica digital cadastral de alta precisão — em escala 1:1.000 na área urbana e 1:5.000 na área rural — incluindo ortofotos, modelo digital do terreno, rede geodésica com centenas de marcos e uma base detalhada de imóveis e logradouros. Esse conjunto consolidou a modernização da cartografia municipal e estabeleceu o alicerce para o avanço do geoprocessamento na cidade.

Com a evolução das necessidades de gestão e planejamento urbano, foi instituído o SIG-SP (Sistema de Informações Geográficas do Município de São Paulo), regulamentado pelo Decreto nº 57.770/2017. O sistema foi concebido como um ambiente integrador, capaz de reunir dados de diversas áreas — urbanísticas, sociais, ambientais, econômicas e administrativas — sem a necessidade de centralizar todas as bases em um único repositório. Essa abordagem permitiu a criação de um ecossistema dinâmico, onde diferentes sistemas se conectam e compartilham informações georreferenciadas de forma padronizada, seguindo diretrizes do Open Geospatial Consortium (OGC).

O GeoSampa

O portal GeoSampa surge, em 2014, como a principal interface pública desse sistema. O portal disponibiliza, de forma aberta e acessível, centenas de camadas temáticas - que incluem desde informações cadastrais e zoneamento até equipamentos públicos, áreas de risco e a localização de aproximadamente 650 mil árvores do sistema viário. Além da visualização interativa, os usuários podem realizar downloads em diversos formatos (como SHP, KMZ, DXF, PDF e JPG), promovendo transparência, apoio à pesquisa, inovação e participação social. O crescimento do uso ao longo dos anos — chegando a milhões de acessos anuais — demonstra a relevância do portal tanto para a administração pública quanto para a sociedade.

A evolução do GeoSampa e do SIG-SP também passou por importantes etapas de modernização tecnológica. Foram realizadas atualizações nas versões do GeoServer, ampliação da capacidade dos servidores para lidar com grandes volumes de dados, atualização das ferramentas de ETL (extração, transformação e carga de dados) e adoção de novas soluções para o desenvolvimento do frontend, com destaque para o uso do Giframework. Todas essas melhorias foram baseadas em tecnologias open source, reforçando a estratégia de sustentabilidade, interoperabilidade e independência tecnológica do município.

A modernização do frontend, em especial, trouxe ganhos significativos na experiência do usuário, com interfaces mais intuitivas, melhor desempenho na navegação e maior capacidade de integração com diferentes serviços e bases de dados e a otimização da navegação pelo celular. Isso tornou o acesso às informações geoespaciais mais ágil e eficiente, ampliando o alcance do portal e facilitando seu uso por públicos diversos — desde técnicos especializados até cidadãos em geral.

Projeto SIG-SP

O SIG-SP é o instrumento para a produção, manutenção, análise, disseminação e divulgação de informações mapeadas como subsídios para o planejamento, implementação e gestão de políticas públicas nas áreas de atuação do governo municipal, priorizando o atendimento ao cidadão. O sistema é constituído, de forma integrada, pelos subsistemas de pessoas, procedimentos organizacionais, dados, programas e equipamentos computacionais, abrangendo os dados sociais, culturais, econômicos, financeiros, patrimoniais, urbanísticos, administrativos, físico-territoriais, ambientais, geológicos, geográficos, cartográficos e outros de interesse do Município, sendo, essencialmente, um sistema aberto, dinâmico e permanente, em constante adequação à realidade urbana. Nesses termos e diante do grande volume de informações disponíveis na administração municipal, a Prefeitura de São Paulo optou pela adoção de uma visão integradora. Ao invés da construção de um novo sistema ou mesmo da migração de todos os dados para uma base única, optou-se pela criação de um ambiente compartilhado que integre dados geográficos de referência (distritos, quadras fiscais, cartografia, por exemplo) aos sistemas de informações já existentes, permitindo a visualização das informações oriundas destes diretamente no mapa. A disponibilização de um volume de informações de aproximadamente 152 terabytes a uma grande quantidade de usuários representa um desafio, principalmente, se consideradas as questões como integridade e segurança dos dados. O projeto de construção do SIG-SP vem sendo implantado passo a passo, de modo a permitir a disponibilização de ferramentas aos usuários já nos primeiros meses de projeto. Como principais etapas, destacam-se

  • A construção da base corporativa de dados pela organização dos dados cartográficos (Mapa Digital da Cidade), cadastrais (setor, quadra, lotes, logradouros) e temáticos;
  • A implantação de um sistema de infraestrutura dos dados municipais;
  • Elaboração do catálogo de metadados;
  • Integração das bases e sistemas já existentes via conversão de dados espaciais.

Para garantir a sustentabilidade do projeto, foram utilizadas geotecnologias compatíveis com as diretrizes nacionais de dados espaciais e ainda tecnologias GEOSERVER / OpenLayers para a construção da Infraestrutura de Dados Espaciais, além do portal de mapas. Trata-se do primeiro projeto de geoprocessamento na Prefeitura de São Paulo a usar tecnologia de software livre. Dentre os requisitos técnicos de implantação, vale destacar a definição de padrões de intercâmbio de dados geográficos, compatível com a Open Geospatial Consortium - OGC, concretizando uma visão voltada para a transparência, compartilhamento que garanta a operabilidade e o efetivo uso das informações geoespaciais.
